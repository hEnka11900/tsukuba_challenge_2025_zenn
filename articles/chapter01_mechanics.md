# まえがき

始まりました，つくばチャレンジ備忘録!

我々，チーム三人行がどのようなフレームワークで，ロボットを作成していったかを述べていきます．1本目の記事では，主に機械設計について述べていきます．今後のつくばチャレンジ出場に活かしていただけると幸いです．




# ハードウェアに求められる要件

最も重視すべきは，故障しないハードウェアを設計することです．ありがたいことに私が製作したSalamander-2は，目立った故障なく大会を終えることができました．
故障しないハードウェアを製作すれば，上流のSLAM, navigation等の自律移動にかける時間を長くすることができますが，図らずも故障しやすいハードウェアができてしまうと，ソフトウェア担当者が作業できる時間が減ってしまいます．






そのために私が意識して設計したポイントを述べます．

- 荷重を受けるすべての部品を金属にする
- 可能な限りシンプルな構造にする
- あとから追加装備を積めるような柔軟な設計をする
- 配線をいじりやすい設計にする



# コースによる制約

つくばチャレンジコースには，屋内ロボットで求められないような仕様が要求されます．これらをハードウェアで先に解決しておくことで，ソフトウェア担当者が作業できる時間が増えます．


### 1.雨天対策


雨天でも実験が行える耐候性


### 2.芝生・坂道・縁石対策

で登りきれるトルク，タイヤ径


弊チームは，navigationはうまく行っているが，市役所の坂道を登りきれずに調整する時間が増えたことで，結果的に実験に費やせる時間が減りました．



### 3.信号対策

で余裕を持って渡り切る速度を出せるモータ

### 4.バッテリ対策

実験走行を乗り切れるだけのバッテリ 

(2025年度の実験走行は10:00~16:00の6時間)




# 大会規定による制約

つくばチャレンジ実行委員会によって規定される仕様がいくつかあります．基本的に出場する年のルールに準拠して設計しましょう．


| 仕様 | 備考 |
| ---- | ---- |
| 大きさ | 小型ロボットにとって特に最低地上高(2025年度は60cm)が厳しいです．LiDARを高い位置に置く等で対応しましょう |
| 重量 | 重量は市販品の大型ロボットを使わない限りそこまで気にならないでしょう |
| 標章・旗 | 自律移動・手動走行などの旗をつける必要があります．つける場所には規定があるので，つける場所をあけておきましょう |
| ナンバープレート | こちらには指定のサイズで印刷する必要があり，大きいので小型ロボットはナンバープレートをつける場所も込みで設計しましょう |
| 最大速度 | 6km/hと決められています．MDに最大回転数を設定する等してオーバーしないようにしましょう | 
| クッション | ロボットがぶつかる可能性を考慮して，予めRを大きくつけておくか，市販のコーナーガードを取り付けましょう．走行前に車検があり，危険な角があるロボットは走行できない規定になっています |
